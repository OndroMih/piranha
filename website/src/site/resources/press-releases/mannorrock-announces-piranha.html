Manorrock.com, an OSS group, has introduced Piranha as a new Java based cloud runtime. The name was chosen in recognition of a Piranha fish being able to furiously eat its food, which translates here to see how many layers of a server can be stripped away and still have something usable.

The technology can be used to run server oriented Java libraries, specifically those making direct or indirect use of the Jakarta Servlet APIs, which include Jakarta EE and MicroProfile, but also frameworks like Spring.

Piranha incorporates technology for scaling the programming model up and down, with the most scaled down version primarily suitable for inclusion in serverless functions, and the most scaled up version suitable for standalone server use.

This first public iteration, Piranha 19.12.0, passes the MicroProfile JWT TCK, which explicitly certifies it as a MP JWT compatible implementation, and implicitly attests to a large collection of underlying technologies working. Plans call for Piranha to pass additional TCKs over time. Its current focus is on passing the Jakarta Servlet TCK.

Piranha provides integration for components developed outside the Piranha project, which include components developed by OmniFaces for Jakarta Authentication, Jakarta Authentication and MicroProfile JWT.

Availability
The first public build of Piranha is available as of now as a free, community downloadable project.
